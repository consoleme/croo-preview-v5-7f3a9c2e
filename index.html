Croo Web Features

Modes for how developers actually work.

Different tasks need different presentation layers. Croo can show raw terminal reality, quiet summaries, or a guided conversational surface without forcing one mode all day.

Discussion mode

Two or three agents can debate, delegate, revise, and converge while Croo tracks roles, turns, majority/full consensus, backchannel handoffs, decisions, and next actions.

Presentation layers

Original for raw panes, Alpha for concise signal, Beta for guided work, and Gamma for a more human conversational surface.

Notes and memory

Project notes, lessons, handoffs, and operator context live beside the agents so the fleet can learn from the workspace correctly.

Live model controls

Switch model, effort, target host, macros, and session commands from the pane menu without rebuilding the workspace.

Multi-project fleets

Run several products, repos, or servers simultaneously while each project keeps its own roster, state, notes, and Work Map.

Resumable agents

Agents can run on different servers, be brought into one fleet, survive reboots, and resume work without losing their project identity.

Reminders

Scheduled prompts, quiet hours, alert routing, recurring checks, and voice wording stay editable instead of becoming hard-coded product behavior.

Alerts and cues

Configure what gets spoken, what appears as text, what reaches Telegram, and what stays quiet in croo-web.

Screensavers

Idle surfaces can become ambient fleet dashboards, Work Map walls, progress views, or company-specific displays.

Reusable prompts and live model switching.

Croo turns operator habits into fast controls: run review and handoff macros, refresh role context, teach croo-awareness, change model or effort, and retarget an agent while work is moving.

  • Macros become configurable prompt packs for review, handoff, triage, release, or team rituals.
  • Model, effort, and target can change from the control surface as the job changes.
  • Session commands stay close: status, help, clear, terminate, orient, and recover.
Configurable signals for real operators.

Croo treats alerts and cues as part of the operating surface: voice, text, Telegram, sounds, quiet modes, and per-event behavior are configurable.

  • Speak or show text for agent needs-input, stuck states, overnight jobs, and custom events.
  • Route important alerts to Telegram while keeping desk-only cues local.
  • Tune urgency, quiet periods, chimes, voice wording, and popup behavior.
